Web Tools give Tasktronaut the ability to fetch content from specific URLs during your tasks. This is useful when you already know the documentation page, changelog, or reference URL you want to inspect.

<Warning>
	Web Tools require the **Cline provider**. They are not available when using other providers like OpenRouter, Anthropic, AWS Bedrock, etc.
</Warning>

## How Web Tools Work

Tasktronaut currently exposes one built-in web tool:

- **web_fetch**: Fetches and analyzes content from a specific URL

When Tasktronaut determines that a known URL would help complete your task, it can use `web_fetch` to retrieve and analyze that page.

## Enabling Web Tools

Web Tools are available when using the Cline provider. To use them:

1. Make sure you're signed in to Cline
2. Ensure you're using the Cline provider
3. Enable the Web Tools toggle in the Feature Settings menu

<Note>
	Web tools can be auto-approved using the "Fetch external URLs" setting in [Auto Approve](/features/auto-approve). In Tasktronaut's current product boundary, that setting governs built-in `web_fetch`, not a general browser automation capability.
</Note>

## Use Cases

### Looking Up Documentation

When working with unfamiliar libraries or APIs:
- Fetch specific official documentation pages
- Retrieve API reference pages
- Inspect examples and usage patterns from a known URL

### Research Before Implementation

Before implementing a feature:
- Fetch design docs, changelogs, or issue threads you already have links to
- Inspect known issues or limitations from specific project pages

### Checking Latest Information

For time-sensitive information:
- Latest release notes and changelogs from known URLs
- Recent bug fixes or security updates from official project pages
- Current recommended versions when documented on official sources
