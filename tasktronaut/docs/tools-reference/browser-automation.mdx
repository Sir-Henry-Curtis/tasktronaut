Tasktronaut's baseline product does not ship general browser automation. Instead, it supports controlled external URL retrieval through `web_fetch` and approved research MCP tools.

## Capabilities

- Retrieve content from a specific URL
- Inspect official documentation and public reference pages
- Pull external text into the task context for analysis
- Combine URL retrieval with approved MCP research tools when policy allows

## Basic Commands

Ask Cline to fetch a specific URL with simple instructions:

```text
Fetch https://example.com and summarize the page
```

```text
Fetch https://example.com/docs/install and tell me the install steps
```

```text
Fetch the changelog at https://example.com/changelog and list the latest release notes
```

## Example Workflows

### Inspecting Official Documentation

```text
Fetch https://example.com/docs/api and summarize the authentication requirements
```

### Reviewing Release Notes

```text
Fetch https://example.com/releases and list the latest compatibility changes
```

## Important Details

### No General Browser Session

Tasktronaut does not maintain a general-purpose interactive browser session in its baseline profile. If you need multi-step website interaction, use an approved MCP/browser integration explicitly.

### Prefer Official Sources

For consistency and provenance:

- prefer official docs, changelogs, package pages, and vendor-hosted references
- avoid ad hoc search unless policy-approved external research tools are available
- record provenance when external findings affect project decisions

## Common Use Cases

- **Documentation Review**: Inspect official docs and API references
- **Release Research**: Read changelogs and release notes
- **Reference Gathering**: Pull exact external URLs into the task for analysis
- **Controlled External Context**: Add bounded outside information without a general browser surface

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Website doesn't load | Use the full URL with `http://` or `https://` |
| Content is too noisy | Prefer a more specific documentation or release-notes URL |
| You need interaction, not retrieval | Use an approved MCP/browser integration rather than baseline `web_fetch` |
